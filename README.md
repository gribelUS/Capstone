# Capstone Traffic Controller

## How to Run on Jetson Nano

### Option 1: Internal Simulation
The controller runs its own built-in Simulation

```bash
python controller.py --simulate --sim-pattern balanced
```

| Flag | Description |
|------|-------------|
| `--simulate` | Tells controller to use simulation instead of camera |
| `--sim-pattern` | Traffic pattern: balanced, morning_rush, evening_rush, east_west_heavy, north_south_heavy, light_traffic, heavy_traffic |

### Option 2: External Simulation Server
Run a separate simulation that the controller connects to.

Terminal 1: Start the simulation Server
```bash
python simulation.py --mode server --port 5556 --pattern balanced
```

Terminal 2: Start the controller
```bash
python controller.py --simulate --sim-port 5556
```

### Option 3: SUMO Traffic Simulation
Use SUMO (Simulation of Urban Mobility) for realistic traffic simulation with GUI visualization.

**Prerequisites:**
```bash
pip install traci sumolib
```

**Run with SUMO:**
```bash
python controller.py --simulate --sim-type sumo --sumo-demand balanced
```

| Flag | Description |
|------|-------------|
| `--simulate` | Tells controller to use simulation instead of camera |
| `--sim-type` | Simulation type: internal or sumo |
| `--sumo-demand` | SUMO traffic pattern: balanced, morning_rush, evening_rush, east_west_heavy, north_south_heavy, light_traffic |

**SUMO GUI:** When using `--sim-type sumo`, the SUMO GUI will open showing:
- Real-time vehicle movement through the intersection
- Traffic light states (green/yellow/red)
- Vehicle queues at each approach

### Option 4: Test with both CV and Simulation
Terminal 1: Start CV
```bash
cd "CVModule/CV Module"
python yolo_test.py --video video.mp4 --port 5555
```

Terminal 2: Start Controller connected to the real camera
```bash
python controller.py --cv-port 5555
```

## Data Logging

The system automatically logs all traffic control data to CSV files in the `logs/` directory.

### Log File Location
- Directory: `logs/`
- Filename format: `traffic_log_YYYYMMDD_HHMMSS.csv`

### Logged Data Columns

| Column | Description |
|--------|-------------|
| `timestamp` | ISO format timestamp of the control cycle |
| `phase` | Current traffic light phase (e.g., PHASE_1_N_S_THRU) |
| `duration_s` | Duration (seconds) since current phase started |
| `decision` | Controller decision: HOLD, CHANGE, EXTEND |
| `reason` | Reason for decision: min_green_not_met, max_green_reached, gap_out, other_phase_demand, etc. |
| `N_thru_count` | Vehicle count on North through lanes (det_N_thru_1 + det_N_thru_2) |
| `S_thru_count` | Vehicle count on South through lanes (det_S_thru_1 + det_S_thru_2) |
| `E_thru_count` | Vehicle count on East through lanes (det_E_thru_1 + det_E_thru_2) |
| `W_thru_count` | Vehicle count on West through lanes (det_W_thru_1 + det_W_thru_2) |
| `N_left_count` | Vehicle count on North left turn lane |
| `S_left_count` | Vehicle count on South left turn lane |
| `E_left_count` | Vehicle count on East left turn lane |
| `W_left_count` | Vehicle count on West left turn lane |

### How Vehicle Counts Are Measured

#### Internal Simulation Mode
Vehicle counts are generated by the built-in `TrafficSimulation` class:
- **Arrival Model**: Probabilistic arrival based on traffic pattern (vehicles per hour)
- **Departure Model**: Vehicles depart when their approach has a green light
- **Queue Tracking**: Each detector lane maintains a queue of vehicles with arrival times
- **Count Calculation**: `count = number of vehicles currently waiting at detector`

#### SUMO Simulation Mode
Vehicle counts come from SUMO's microscopic simulation:
- **Detection Method**: Inductive loop simulation at stop lines
- **Count**: Number of vehicles currently within the detector zone
- **Wait Time**: SUMO's built-in waiting time calculation for each vehicle

#### CV Module Mode (Real Camera)
Vehicle counts come from YOLO object detection:
- **Detection**: YOLO v11 identifies vehicles in video frames
- **Tracking**: Vehicles tracked across zones using unique IDs
- **Count**: Vehicles currently in each detection zone
- **Zone Mapping**: Video zones mapped to detector IDs via `cv_to_detector_map` in rules.json

### Traffic Pattern Definitions

| Pattern | Description | N Through | S Through | E Through | W Through | Left Turns |
|---------|-------------|-----------|-----------|-----------|-----------|------------|
| balanced | Equal traffic all directions | 300/hr | 300/hr | 300/hr | 300/hr | 100/hr |
| morning_rush | Heavy north-south | 600/hr | 200/hr | 200/hr | 200/hr | 150 N/S, 50 E/W |
| evening_rush | Heavy south-north | 200/hr | 600/hr | 200/hr | 200/hr | 50 N/S, 150 E/W |
| east_west_heavy | Heavy east-west | 100/hr | 100/hr | 500/hr | 500/hr | 50 each |
| north_south_heavy | Heavy north-south | 500/hr | 500/hr | 100/hr | 100/hr | 150 each |
| light_traffic | Minimal traffic | 100/hr | 100/hr | 100/hr | 100/hr | 20 each |

### Interpreting the Log Data

**Example Analysis:**

```csv
timestamp,phase,duration_s,decision,reason,N_thru_count,S_thru_count,...
2024-02-18T02:02:55.123456,PHASE_1_N_S_THRU,15.2,HOLD,min_green_not_met,5,3,...
2024-02-18T02:02:57.234567,PHASE_1_N_S_THRU,17.3,EXTEND,continuing,4,2,...
```

- **HOLD + min_green_not_met**: Phase is still in minimum green time, cannot change yet
- **EXTEND + continuing**: Traffic detected, extending green time up to max
- **CHANGE + other_phase_demand**: Queue cleared or other direction has waiting traffic

### Performance Metrics (Derived from Logs)

From the logged data, you can calculate:

1. **Average Delay**: Mean of `duration_s` for all vehicles
2. **Queue Length**: Sum of all approach counts during red phases
3. **Phase Distribution**: Percentage of time spent in each phase
4. **Cycle Length**: Time between consecutive phase changes
5. **Level of Service**: Based on delay per vehicle (A-F scale)

## SUMO Network Details

The SUMO simulation models the Gale Catlett Dr / Patteson Drive intersection in Morgantown, WV.

### Network Configuration

- **4-legged intersection** with dedicated turn lanes
- **2 through lanes** per direction
- **1 left turn lane** per direction
- **Total of 12 detector zones** matching the controller's zones

### Detector Mapping

| SUMO Lane | Detector ID | Approach |
|-----------|-------------|----------|
| N_thru_1 | det_N_thru_1 | North Through Lane 1 |
| N_thru_2 | det_N_thru_2 | North Through Lane 2 |
| N_left | det_N_left | North Left Turn |
| S_thru_1 | det_S_thru_1 | South Through Lane 1 |
| S_thru_2 | det_S_thru_2 | South Through Lane 2 |
| S_left | det_S_left | South Left Turn |
| E_thru_1 | det_E_thru_1 | East Through Lane 1 |
| E_thru_2 | det_E_thru_2 | East Through Lane 2 |
| E_left | det_E_left | East Left Turn |
| W_thru_1 | det_W_thru_1 | West Through Lane 1 |
| W_thru_2 | det_W_thru_2 | West Through Lane 2 |
| W_left | det_W_left | West Left Turn |

### Traffic Light Phases

The controller synchronizes with SUMO's traffic lights:

| Controller Phase | SUMO Phase Index | Description |
|-----------------|------------------|-------------|
| PHASE_1_N_S_THRU | 0 | North-South Through Green |
| PHASE_2_N_S_LEFT | 2 | North-South Left Turn Green |
| PHASE_3_E_W_THRU | 4 | East-West Through Green |
| PHASE_4_E_W_LEFT | 6 | East-West Left Turn Green |
